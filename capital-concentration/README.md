# 资金集中度看板

**两次自动更新：**
- A股收盘后（北京时间 16:00）
- 美股收盘后（北京时间次日 06:00，UTC 22:00）

## 看板内容

| 模块 | 说明 |
|---|---|
| 综合分 + 历史走势 | 资金集中度0-100，历史分位数加权 |
| A股指数 | 科创50/上证/深证/沪深300/中证1000/2000/芯片 |
| 全球市场 | 标普/纳斯达克/SOX半导体/KOSPI/三星/SK海力士/特斯拉/英伟达 |
| 半导体成交额占比 历史 | 科创50内半导体股票的成交额集中度走势 |
| QVIX历史 | 科创/创业/50ETF/300ETF/500ETF/中证1000 期权波动率多线对比 |
| 期权PCR | Put/Call Ratio成交量口径历史，判断市场情绪方向 |
| 两融余额动量 历史 | 杠杆资金进出速度 |
| 今日板块资金流 | 行业净流入排名（前5流入 / 前5流出）含航天/半导体等 |
| 利率历史 | 美联储 + 日本央行历史决议利率对比图，最近24个月 |
| Shiller CAPE | 美股周期调整PE，直链gurufocus |
| CME FedWatch | 提供直达链接，降息概率看官方图最准 |

## 数据源

- **akshare** — 全部A股数据（免费，无需key）
- **yfinance** — 美股/韩股指数
- **gurufocus** — Shiller CAPE（网页抓取）
- **FRED API** — 美联储利率备用（免费key，可选）

## 部署

1. 新建 GitHub 仓库，推入本项目所有文件
2. Settings → Secrets → 添加 `FRED_API_KEY`（可选，没有的话利率走akshare口径）
3. Settings → Pages → Source: GitHub Actions
4. Actions 页面手动触发一次，确认跑通
5. 之后每天自动更新两次

## 关于历史数据

首次跑时 `data/history.jsonl` 为空，综合分显示50（中性）属正常，
QVIX/利率历史本身就有完整历史数据会直接展示，
集中度综合分的历史走势需要积累几周才有意义。

如需一次性回填集中度历史，运行：
```bash
cd scripts
python backfill_history.py --days 30
```
